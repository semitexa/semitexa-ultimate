# AI_ENTRY.md — Project Reference

> Scaffold doc (`semitexa/ultimate`). Edit root → `bin/semitexa scaffold:sync-docs`. Consumer projects: overwritten by `bin/semitexa init --only-docs`.

**`AGENTS.md` is the operator manual** — read it first. This file is project-specific: Golden Path, stack, module layout, generator details.

## Golden Path — before you reach for Read/Grep/Edit

| You want to… | Do this **first** | Fallback |
|---|---|---|
| Classify what was asked | `bin/semitexa ai:task "<prose>" --json` | — |
| Commands that exist | `bin/semitexa ai:ask capabilities --json` | `bin/semitexa` (flat list) |
| Project / module / route / event | `bin/semitexa ai:ask <project\|module\|route\|event> --json` | Read/Grep |
| Who uses class `X` | `bin/semitexa ai:review-graph:query --usages=<FQCN> --json` | Grep |
| Blast radius of a change | `bin/semitexa ai:review-graph:impact <FQCN> --json` | Grep across `packages/**` + `src/**` |
| DI bindings for interface | `bin/semitexa contracts:list --json` + `ai:review-graph:query --usages=<InterfaceFQCN>` | grep `implements` |
| Create a page/endpoint | `bin/semitexa make:page --module=<M> --name=<N> --path=/<p> --method=GET --write --json` | hand-writing payload + handler + resource |
| Create service / contract / listener | `bin/semitexa make:<service\|contract\|event-listener> --write --json` | Edit/Write from scratch |
| Verify a change | `bin/semitexa ai:verify --files=<paths> --json` | running phpunit by hand |
| Decompose non-trivial work | `bin/semitexa ai:epic start` + `ai:work start` | keeping plan only in chat |
| Read logs | `bin/semitexa ai:ask logs --grep=<term> --lines=200 --json` | `tail var/log/app.log` |

**If the left column covers the question, `Read` / `Grep` is the wrong first move.**

## Three commands every session

1. **`ai:orient`** — session state on cold start.
2. **`ai:ask <subject>`** — introspect instead of reading. One call ≈ 5 file reads.
3. **`ai:verify`** — after every edit. Non-negotiable.

## What not to do

- ❌ Scan `var/docs/**` for backlog questions — that's `ai:epic list` / `ai:work list`.
- ❌ Write a planning/audit/analysis/report `.md` to project root — **all temporary docs go in `var/docs/`**. Root is reserved for framework scaffold docs (AGENTS, AI_ENTRY, AI_CONTEXT, CLAUDE, AGENTS_DOCTRINE, AI_REFERENCE, README, AI_NOTES).
- ❌ Hand-write a payload/handler/resource trio — `make:page` exists.
- ❌ Use `Grep` for structural questions about classes — use `ai:review-graph:*`.
- ❌ Treat the 75-command `bin/semitexa` listing as the capability surface — use `ai:ask capabilities`.
- ❌ Assume `ai:task` is useless because it returned `recipe=unknown_task` — that is the *correct* signal to proceed without a recipe.

## Stack

- **PHP** `^8.4`; Swoole; `semitexa/core` dev-main or v1.x
- **Symfony** 7.x, **Twig** ^3.10, PSR Container + Semitexa DI (`#[SatisfiesServiceContract]`, `#[InjectAsReadonly|Mutable|Factory]`)
- Exact pins in `composer.lock`. **No Laravel / Illuminate.**

## Module layout

```
src/modules/<Module>/
├── Application/ {Payload, Resource, Handler/PayloadHandler, View/templates}
└── Domain/      {Service, Contract, Event, Model}
```

- **New routes go inside a module** (`src/modules/<M>/`), never in project `src/`. `App\` is not route-discovered.
- **No per-module PSR-4 entries in root `composer.json`** — modules autoload from `src/modules/`.
- **Canonical docs:** `packages/semitexa-docs/` (public) and `packages/<pkg>/docs/` (package-local). `var/docs/` is scratch. Root `./docs/` is **not** official.
- **Generated files** carry `/** @generated by 'semitexa make:X' */` — edit directly, do not regenerate to overwrite.

## Generators (`make:*`)

All are **dry-run by default** — pass `--write` to commit. All emit `next_command` for what to run next.

| Generator | Output |
|---|---|
| `make:page` | Payload + Handler + Resource + template in one shot — **use this first** |
| `make:payload` | Single request DTO with route + validation attributes |
| `make:handler` | Handler bound to Payload and Resource |
| `make:resource` | Response class with optional Twig template |
| `make:service` | Injectable service with `#[AsService]` |
| `make:contract` | Interface + implementation with `#[SatisfiesServiceContract]` |
| `make:event-listener` | Listener with `#[AsEventListener]` |
| `make:command` | CLI command with `#[AsCommand]` |
| `make:module` | Full module directory (10 dirs) |
